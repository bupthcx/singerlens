## What This Project Is About

The starting point was **SingerLens**, an explainable AI-cover detector based on singing-style features such as pitch, HNR, vibrato regularity, and stability.

The project then evolved into **SVDD-Audit**: an audit of whether singing voice deepfake detectors really learn "AI cover-ness", or whether they rely on shortcut cues such as:

- generation/vocoder family,
- production pipeline artifacts,
- bandwidth and codec traces,
- singer or dataset identity.

The main finding is that standard in-domain evaluation can be highly optimistic. Detectors that look near-perfect inside one dataset often collapse under unseen generators, unseen vocoders, or cross-dataset transfer.

## Key Results

- **Cross-dataset collapse:** in-domain AUC can be around 0.90-0.99, while cross-dataset AUC drops near chance for RF/AASIST/SSL baselines.
- **Generation-family confounds:** LOVO/LOGO and factorial controls show detectors depend heavily on seen vocoder/generator families.
- **Not just vocoder detection:** copy-synthesis and factorial controls show the shortcut is multi-axis: vocoder family, generator paradigm, and production signature.
- **Bandwidth/production signature:** low-pass and resampling controls can make real vocals look fake to the detector.
- **Unsupervised fixes are weak:** domain adaptation and confound removal mostly fail.
- **Minimal target supervision works:** MSA/U-MSA provides a practical, label-efficient exit, but it remains target-domain supervision.
