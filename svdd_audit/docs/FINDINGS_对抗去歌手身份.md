# 实验发现 IX：对抗式去歌手身份(DANN)——诚实的负面结果

## 方法
Domain-Adversarial(DANN+梯度反转层GRL):编码器(14 CLEAN特征->32->16)+真伪头+歌手判别头(经GRL)。
训练使真伪头分对、歌手判别头分不出 -> 表示对歌手不变。lambda按 2/(1+e^-10p)-1 调度。
评估 Leave-One-Singer-Out,5种子平均;对比普通MLP(lambda=0,无对抗)。环境qwen3-asr torch2.5。

## 结果(LOSO 平均 AUC, 5种子)
| test on | 普通MLP | DANN |
|---|---|---|
| singer_a | 0.610 | 0.595 |
| singer_b | 0.609 | 0.616 |
| singer_c | 0.801 | 0.771 |
| MEAN | 0.673 | 0.661 (ΔAUC -0.012) |

对照: RF+raw=0.679, RF+per-singer-z=0.745。

## 结论:对抗式去歌手身份未提升,反而略降
原因分析:
1. 仅2个源domain(训练歌手):域对抗需多domain才能学到可泛化的不变性,2个约束太弱,
   学到的'对2歌手不变'无法迁移到第3个未见歌手。
2. 小数据(~340训练样本):对抗训练不稳定、数据饥渴,小MLP易过拟合。
3. 特征高度纠缠:去歌手信息时连带去掉与真伪相关信息,真伪头受损。

## 核心结论
在'少domain+小数据'场景,简单的 per-singer 标准化(0.745)显著优于对抗DANN(0.661)。
推荐 per-singer-z 作为去歌手身份的方法;DANN作为'更多歌手/数据时的未来工作'。
(这是常见且诚实的实务发现:简单归一化 > 复杂对抗,在低资源时。)

## 补充:扩到6歌手(5训练domain)后重跑
为检验'domain太少'假设,数据扩到6歌手(单依纯/孙燕姿/邓紫棋/林俊杰/王菲/徐佳莹,含男声/空灵/民谣),1085片段。
| 方法 | LOSO平均AUC(6歌手) |
|---|---|
| 普通MLP | 0.716 |
| DANN | 0.714 (ΔAUC -0.002) |
| RF+per-singer-z | 0.765 |
结论:即便5个训练domain,DANN仍≈普通MLP且双双低于per-singer-z。
说明问题不只是domain少——对抗式复杂方法在低维表格+小数据任务上不如简单per-singer归一化。
'简单赢复杂'结论强化。普通MLP随歌手增多确有提升(0.673->0.716),但对抗分量无贡献。
