# 带宽连续扫描:误判临界点 (FINDINGS_bandwidth_sweep.md)

## 目的
Exp3 用 4k/3k 两个离散点显示"带限把真唱翻成 fake"。本实验做**连续带宽扫描**,画"真唱伪分数 vs 低通截止
频率"曲线,定位**误判临界点**——一个适合现场展示"过了这个频率系统就指鹿为马"的图。脚本 `bandwidth_sweep.py`。

## 设置
- 200 个 SingerLens 真唱,逐个低通到一系列截止频率,re-extract FULL,用 SingerLens 域内检测器打分。
- **音频 16kHz(奈奎斯特 8kHz)**,故 none/≥8k = no-op(切不掉 8k 以上)。扫描有效区间 ≤8k:
  none(16k) / 7.5k / 6k / 5k / 4k / 3.5k / 3k / 2.5k / 2k。

## 结果(真唱 mean fake-score / flip→fake)

| 截止 | mean_fake_score | flip→fake |
|---|---:|---:|
| 16k(无滤波) | 0.085 | 0% |
| 7.5k | 0.206 | 2.5% |
| **6k** | **0.414** | **32.5%** |
| 5k | 0.401 | 27.0% |
| 4k | 0.397 | 13.0% |
| 3.5k | 0.429 | 30.5% |
| **3k** | **0.450** | **46.5%(峰)** |
| 2.5k | 0.375 | 14.0% |
| 2k | 0.335 | 5.0% |

## 关键发现
1. **存在明显的误判临界点 ≈ 6–7 kHz**:从 7.5k(0.21,flip 2.5%)到 6k(0.41,flip 32.5%)是一次**陡跳**
   ——真唱伪分数翻倍、flip 率涨 13 倍。一旦带宽跌破 ~7kHz(即损失 ~7kHz 以上内容),检测器开始把真唱判为 fake。
2. **3–6 kHz 是"危险带"**:flip 率 27–46%,在 3k 处达峰(46.5%)。bilibili/平台压缩与人声分离常把带宽压到
   这个区间,正对应野生真唱的高假正风险。
3. **极端截止(2–2.5 kHz)分数回落**(flip 14%→5%):过度退化后进入另一频谱区,不再"恰好像 fake"——
   说明检测器的"fake 区"是一段**中等带宽签名**,非"越窄越假"的单调关系(诚实 nuance)。
4. **机制一致**:曲线是 5.4 cleanliness-axis(频谱带宽/编解码签名)的连续因果证据;非单调性进一步说明
   它锁定的是特定带宽足迹而非简单"脏化"。

## 用途
图 `bandwidth_sweep.png`(伪分数 + flip 率 vs 截止频率)适合作论文 5.4 因果证据强化图 / 现场 demo:
**"过了 ~6–7kHz 临界点,系统就开始把真人误判为 AI。"**

## 限制
- 受控合成低通(非真实平台转码);绝对 flip 率不可外推。
- 单检测器(SingerLens FULL)+ 单语料真唱;非单调性的精确成因(MFCC 进入哪一区)未深究。
